# 最新资讯模块开发记录

> 日期：2026-05-13  
> 涉及范围：首页资讯模块重构 + 最新资讯列表页 + 文章详情页

---

## 一、需求概述

### 首页最新资讯模块
- 横向展示 4 个卡片，每个卡片使用文章缩略图作为背景图
- 卡片下方显示文章标题（叠加在背景上）
- 卡片点击跳转至文章详情页 `article-detail.html?id=xxx`
- 模块底部有"查看更多"按钮，跳转到最新资讯列表页 `articles.html`

### 最新资讯列表页（`articles.html`）
- 每行显示 5 个大卡片，缩略图背景 + 标题叠加
- 点击卡片进入文章详情页
- 静态页面，留接口占位和注释

### 文章详情页（`article-detail.html`）
- 显示文章标题、作者、发布时间、封面大图、富文本正文
- 页面内容为静态，留接口占位和注释

### 接口数据字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int(10) | 文章ID |
| `title` | varchar(200) | 标题 |
| `seo_keywords` | varchar(255) | SEO关键词 |
| `content` | longtext | 文章内容（支持富文本、图片） |
| `author` | varchar(50) | 作者 |
| `create_time` | datetime | 发布时间 |
| `update_time` | datetime | 更新时间 |
| `site_id` | int(10) | 网站ID |
| `region_id` | int(10) | 地区ID，关联 ggh_type |
| `thumbnail` | varchar(500) | 缩略图URL |
| `status` | tinyint(1) | 状态：0 草稿，1 已发布，2 下架 |

### SEO & URL 规则
- 页面和文件命名禁止中文，中文使用拼音表示
- 每篇文章生成 SEO meta 标签（标题 + SEO 关键词）
- 详情页含 JSON-LD Article 结构化数据

## 四、页面结构说明

### 4.1 首页最新资讯模块（`index.html`）

```
<section class="section section-news" id="articleList">
  ├── .section-title（标题：最新资讯）
  ├── .news-grid#newsGrid（4列网格）
  │   └── <a class="news-card">（缩略图背景 + 渐变遮罩 + 标题叠加）
  ├── #newsLoading（加载动画）
  └── .news-more-wrap > .news-more-btn（查看更多 → articles.html）
```

**JS 调用链**：`loadArticles()` → `fetchArticles()`（取4条）→ `renderNewsCards(list)`

### 4.2 最新资讯列表页（`articles.html`）

```
<main class="articles-page-wrap">
  ├── .articles-breadcrumb（面包屑：首页 > 最新资讯）
  ├── .articles-page-header（页面标题 + 描述）
  ├── .articles-page-grid#articlesGrid（5列网格）
  │   └── <a class="articles-page-card">（缩略图背景 + 标题叠加）
  ├── #articlesLoading（加载动画）
  └── .articles-pagination#articlesPagination（分页占位）
```

**JS 调用链**：`loadArticlesPage()` → `fetchArticlesList(page, pageSize)` → `renderArticleCards(list)`

### 4.3 文章详情页（`article-detail.html`）

```
<main class="article-detail-wrap">
  ├── .article-detail-breadcrumb（面包屑：首页 > 最新资讯 > 文章标题）
  └── .article-detail-card
      ├── h1（文章标题）
      ├── .article-detail-meta（作者 · 发布时间）
      ├── .article-detail-cover > img（封面大图）
      ├── .article-detail-content（富文本正文）
      ├── .article-detail-update（更新时间）
      └── .article-detail-back > .article-back-btn（返回列表）
```

**JS 调用链**：`loadArticleDetail()` → `fetchArticleDetail(id)` → `renderArticleDetail(data)` + `updateSEO(data)`

## 六、接口占位说明

### 列表接口

```js
// GET /api/articles?type=news&limit=4       （首页，取4条）
// GET /api/articles?type=news&page=1&pageSize=15 （列表页，分页）
```

### 详情接口

```js
// GET /api/articles/detail?id=1
```

当前使用模拟数据（`Promise.resolve`），接入真实 API 时替换 `fetchArticles()` / `fetchArticlesList()` / `fetchArticleDetail()` 中的注释行即可。

## 七、SEO 处理

- 首页资讯模块：静态兜底卡片（`class="static-fallback"`），JS 不可用时仍可展示
- 列表页：静态标题 + meta 标签 + 面包屑导航
- 详情页：
  - 默认 meta 为占位值（title/keywords/description）
  - JS 加载后动态更新 `<title>` / `<meta name="keywords">` / `<meta name="description">`
  - 自动生成 JSON-LD Article 结构化数据
  - 面包屑含文章标题
