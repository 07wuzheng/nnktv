> 生成时间：2026-05-18（合并自 5-15接口文档 + 5-18接口文档 + 前端优化记录）
> 本文档给后端开发人员看，包含：**你需要开发的所有接口**、**每个接口怎么传参怎么返回**、**返回的数据怎么影响页面**、**前端代码怎么改才能接上**

## 一、快速上手（3分钟看懂）

### 你需要做什么？

1. **开发 12 个 API 接口**（详见第二节）
2. **所有图片字段必须返回完整 URL**（以 `https://` 开头，前端通过 `indexOf('http')` 判断是否有效）
3. **门店数据表必须加 `cover` 字段**（分区列表页当前缺少此字段，卡片图片区显示"图片占位"文字）
4. **文章 `content` 字段入库前必须做 XSS 过滤**（前端直接 `innerHTML` 渲染，不过滤等于注入漏洞）

### 前端已经做了什么？

- ✅ 每个 JS 文件顶部加了 `API_BASE` 配置变量，你部署时只改这一处
- ✅ 每个 mock 函数上方加了 `// TODO: 后端对接` 注释，搜索 `TODO` 即可找到所有需要改的地方
- ✅ `district.html` 的内联 JS 已抽离为 `js/district.js`，现在所有页面逻辑都在独立 JS 文件中
- ✅ 搜索功能预留了 `GET /api/search` 接口说明

### API_BASE 怎么用？

每个 JS 文件顶部都有这行：
```js
var API_BASE = '';  // 留空 = 相对路径；生产环境可设为 'https://api.nanningktv.com' 等
```

- 如果前后端同域部署：留空即可，接口地址如 `/api/ktv/featured`
- 如果前后端不同域：改为你的 API 域名，如 `'https://api.nanningktv.com'`，接口地址会自动拼接为 `https://api.nanningktv.com/api/ktv/featured`

## 三、接口详细定义

> 约定：所有接口返回 JSON。`code=200` 表示成功，`data` 为业务数据。
> 域名：`https://www.nanningktv.com`
> 前端所有图片字段优先使用完整 URL（含 `https://`），前端通过 `indexOf('http')` 判断是否有效

### 1. 获取轮播图数据

**有什么用？** 首页顶部大轮播图，自动播放，4秒切换一张。

**接口地址**：`GET /api/banners`

**请求参数**：无

**返回格式**：
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": 1,
      "image": "https://www.nanningktv.com/img/banner1.jpg",
      "alt": "南宁KTV夏季优惠活动",
      "link": "",
      "sort": 1,
      "status": 1
    }
  ]
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 是 | 轮播图 ID |
| image | string | 是 | 图片完整 URL，**必须含 https://**，前端直接赋值给 `<img src>` |
| alt | string | 是 | 图片 alt 文字，SEO 必需 |
| link | string | 否 | 点击跳转链接，空字符串则不跳转 |
| sort | number | 是 | 排序权重，越小越靠前 |
| status | number | 是 | 1=启用 0=禁用，前端过滤 `status===1` |

**前端怎么用**：
- 请求失败时保留 HTML 静态轮播图（不会白屏）
- `image` 有效 → 动态创建 `<img>` + `<a>` 包裹（有 link 时）
- 轮播自动播放间隔 4 秒，支持左右箭头/指示点/触摸滑动

**⚠️ 对页面的影响**：
- `data` 数组长度决定轮播图张数（当前静态有 3 张，API 成功后全部替换）
- `image` 为空或不含 `http` → 该张轮播图不显示图片（白板）

**对应前端代码**：`js/main.js` → `BANNER_API` 变量 + `fetchBanners()` 函数

### 2. 获取网站标题

**有什么用？** 动态修改浏览器标签页标题，用于后台统一管理标题。

**接口地址**：`GET /api/site-title`

**请求参数**：无

**返回格式**：
```json
{
  "title": "南宁商务场KTV_南宁KTV推荐_包厢预订"
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 覆盖 `document.title` |

**前端怎么用**：成功后直接 `document.title = data.title`，失败保留 HTML 原始 `<title>`

**⚠️ 不影响页面布局**：仅修改浏览器标签标题

**对应前端代码**：`js/main.js` → `loadSiteTitle()` 函数

### 3. 获取联系电话

**有什么用？** 全站电话号码统一管理，改一处全站生效。当前硬编码 `19968122123`。

**接口地址**：`GET /api/contact`

**请求参数**：无

**返回格式**：
```json
{
  "phone": "199-6812-2123"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| phone | string | 是 | 联系电话，可含 `-` 分隔 |

**前端怎么用**：遍历所有 `.phone-number` 元素，更新文本和 `href="tel:xxx"`

**⚠️ 对页面的影响**：
- 首页 header 电话、footer 电话、所有包含 `.phone-number` 的位置文本都会被替换
- 当前硬编码电话出现在以下位置（API 成功后自动替换）：

| 位置 | 文件 | 硬编码值 | 替换方式 |
|------|------|---------|---------|
| header 电话 | 所有 HTML | `199-6812-2123` / `19968122123` | `/api/contact` 返回后更新 `.phone-number` |
| footer 电话 | 所有 HTML | `19968122123` | 同上 |
| 详情页 JSON-LD | detail.html | `19968122123` | JS 动态更新 |
| 招聘页联系 | recruitment.html | `199-6812-2123` | `/api/recruit/ktv-princess` 返回 |
| 弹窗默认值 | district.js / detail.js | `19968122123` | 仅作兜底，API 优先 |

**对应前端代码**：`js/main.js` → `loadContact()` 函数

### 4. 获取热门KTV门店

**有什么用？** 首页"热门KTV门店"区块，4列网格，显示8家门店卡片。

**接口地址**：`GET /api/ktv/featured`

**请求参数**：

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| limit | number | 否 | 8 | 返回条数 |

**返回格式**：
```json
{
  "code": 200,
  "data": {
    "list": [
      {
        "id": "1",
        "name": "XX娱乐会所",
        "district": "youxi",
        "address": "南宁市游戏场区XX路XX号",
        "cover": "https://www.nanningktv.com/img/store1.jpg",
        "create_time": "2026-05-14 10:00:00"
      }
    ]
  }
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 门店 ID，用于跳转详情页 `detail.html?district=xxx&id=xxx` |
| name | string | 是 | 门店名称，渲染为卡片标题 `<h3>` |
| district | string | 是 | 分区拼音 slug：`youxi` / `shangwu` / `gaoduanliangfan` / `huachang` |
| address | string | 是 | 门店地址，渲染为卡片副标题 |
| cover | string | 是 | 门店封面图 URL，**必须含 https://**，否则显示灰色占位 |
| create_time | string | 是 | 创建时间，排序用 |

**前端怎么用**：
- `district` 通过映射表转为中文名（游戏场/商务场/高端量贩场/花场），渲染为标签
- `cover` 有效 → `background-image:url(xxx)`，无效 → 灰色占位
- 点击卡片跳转 `detail.html?district={district}&id={id}`
- 数据为空 → 显示"暂无热门门店"

**⚠️ 对页面的影响**：
- `cover` 为空 → **大面积灰色占位，首页视觉效果极差**，强烈建议后端必须返回
- `name` 过长（>15字）→ CSS 自动截断

**对应前端代码**：`js/main.js` → `fetchFeaturedStores()` 函数

| pageSize | number | 否 | 15 | articles | 每页条数 |

**返回格式**：
```json
{
  "list": [
    {
      "id": 1,
      "title": "南宁游戏场KTV新开业优惠活动",
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 是 | 文章 ID，跳转 `article-detail.html?id=xxx` |
| title | string | 是 | 文章标题 |

**⚠️ 对页面的影响**：

### 6. 获取区域封面图

**有什么用？** 首页4个分区卡片的背景图，每个卡片的图 = 该区第一家门店的封面图。

**接口地址**：`GET /api/ktv/first-store-covers`

**请求参数**：无

**返回格式**：
```json
{
  "youxi": { "cover": "https://www.nanningktv.com/img/store1.jpg" },
  "shangwu": { "cover": "https://www.nanningktv.com/img/store2.jpg" },
  "gaoduanliangfan": { "cover": "https://www.nanningktv.com/img/store3.jpg" },
  "huachang": { "cover": "https://www.nanningktv.com/img/store4.jpg" }
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| {slug}.cover | string | 是 | 该分区首家门店的封面图 URL |

**前端怎么用**（当前已注释，待后端接口就绪后取消注释）：
```javascript
// js/main.js → loadDistrictCovers() 函数
// 取消以下注释即可启用：
const data = await fetchJSON(API_BASE + '/api/ktv/first-store-covers');
document.querySelectorAll('.district-home-card[data-district-slug]').forEach(card => {
    const slug = card.getAttribute('data-district-slug');
    if (data[slug] && data[slug].cover) {
        card.style.setProperty('--district-bg', `url('${data[slug].cover}')`);
    }
});
```

**对应前端代码**：`js/main.js` → `loadDistrictCovers()` 函数

### 7. 获取门店详情

**有什么用？** 门店详情页，显示完整的门店信息、图片、套餐、电话。

**接口地址**：`GET /api/ktv/detail`

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 门店 ID |
| district | string | 是 | 分区拼音 slug |

**返回格式**：
```json
{
  "id": "1",
  "title": "游戏场 XX 娱乐场所（1号店）",
  "time": "2026-05-01",
  "address": "游戏场 XX 路 XX 号 XX 大厦",
  "package": "XX 套餐：XX 小时欢唱 + XX 饮品",
  "description": "环境 XX 优雅，音响品质 XX，XX 聚会首选场所。",
  "phone": "19968122123",
  "district": "youxi",
  "images": [
    "https://www.nanningktv.com/img/store1-1.jpg",
    "https://www.nanningktv.com/img/store1-2.jpg",
    "https://www.nanningktv.com/img/store1-3.jpg"
  ]
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 门店 ID |
| title | string | 是 | 门店名称，渲染到 `<h1>` + SEO meta |
| time | string | 否 | 发布日期，渲染到 `<time>` |
| address | string | 是 | 门店地址 |
| package | string | 否 | 套餐信息 |
| description | string | 是 | 门店描述，按 `\n` 分段渲染为 `<p>` |
| phone | string | 是 | 联系电话 |
| district | string | 是 | 分区拼音 slug |
| images | string[] | 是 | 门店图片列表，**每条 URL 必须含 https://** |

**前端怎么用**：
- `images[0]` 有效 → 更新 `og:image`、`twitter:image`、JSON-LD `image`
- `images` 每个元素有效 → `<img>` 标签，无效 → `.img-placeholder` 灰色占位
- `description` 按 `\n` 换行分段为多个 `<p>` 段落
- 全部 SEO meta（title/description/OG/canonical/JSON-LD）由 `updateSEOMeta()` 动态更新

**⚠️ 对页面的影响**：
- `images` 为空 → 3 个灰色占位块，**视觉上大块空白**
- `phone` 字段同时更新 JSON-LD 结构化数据中的 `telephone`

**对应前端代码**：`js/detail.js` → `fetchDetail()` 函数

## 五、图片字段使用规则

### 图片共用逻辑

| 图片字段 | 首页 | 列表页 | 详情页 |
|---------|------|--------|--------|

| 区域 `cover` | 分区卡片 `--district-bg` | — | — |

### 图片规格建议

| 类型 | 建议尺寸 | 格式 | 说明 |
|------|----------|------|------|
| 门店 cover | 640×400px | WebP/JPG | CSS `background-size:cover` 自动裁切 |
| 文章 thumbnail | 800×450px | WebP/JPG | CSS `background-size:cover` 自动裁切 |
| 轮播图 banner | 1200×500px | WebP/JPG | `<img>` 渲染，CSS `width:100%` |
| 区域 cover | 600×400px | WebP/JPG | CSS `background-size:cover` |

### 图片 URL 验证规则

前端对图片 URL 做以下校验：
1. 必须包含 `http`（`item.cover.indexOf('http') !== -1`）
2. 不能包含 `url` 字符串（排除占位符如 `url1.jpg`）
3. 两条都通过 → 渲染图片；否则 → 灰色占位

**所以后端返回的图片 URL 必须以 `https://` 开头。**

### 怎么改？

每个需要改的地方都有 `// TODO: 后端对接` 注释，在 VS Code 中全局搜索 `TODO` 即可找到所有位置。

改法统一为：
1. 取消注释那行 `return fetchJSON(API_BASE + '/api/xxx')`
2. 删除下方 `Promise.resolve({...})` 的 mock 数据
3. 如有需要，调整返回数据的读取路径（如 `res.data.list`）

### district.js 特殊说明

`district.js` 的改动稍微不同，因为它之前是内联在 HTML 中的，现在已经抽离为独立 JS 文件。改动思路：

```javascript
// 原来的 mock 方式：
var ktvData = [];
for (var i = 1; i <= 12; i++) { ktvData.push({...}); }
renderPage(currentPage);
renderPagination(currentPage);

// 改为 API 调用：
function loadKtvList(page) {
    fetchJSON(API_BASE + '/api/ktv/list?district=' + slug + '&page=' + page + '&pageSize=' + perPage)
        .then(function(res) {
            if (!res || !res.data || !res.data.list) return;
            ktvData = res.data.list;
            var totalPages = Math.ceil(res.data.total / perPage);
            renderPage(page);
            renderPagination(page, totalPages);
        })
        .catch(function(err) {
            console.warn('加载门店列表失败:', err);
        });
}
loadKtvList(1);
```
