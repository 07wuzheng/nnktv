# 详情页导航与推荐区优化

> 修改日期：2026-05-12

## 一、需求概述

1. **左侧导航栏**：点击地区不再停留在详情页内切换，而是跳转至 `district.html` 对应地区列表页
2. **右侧门店推荐**：移除上下页分页，固定展示 5 张推荐卡片；每张卡片只包含**标题 + 介绍 + 图片**
3. **数据一致性**：推荐区数据格式与 `district.html` 的 `ktvData` 保持一致，后端传参字段完全相同

---

## 二、修改文件清单

| 文件 | 修改内容 |
|------|----------|
| `detail.html` | 左侧导航 + 移动端标签栏 `href` 改为 `district.html?name=xxx`；右侧推荐区移除分页控件和静态占位卡片 |
| `detail.js` | 重写：左侧导航改为直接跳转；`fetchRecommend` 固定 5 条、排除当前门店；`renderRecommend` 渲染标题+介绍+图片；数据字段对齐 district.html |
| `css/style.css` | 推荐卡片从"背景图+遮罩+白字"改为"左图+右文"横排布局；移除分页器样式；更新响应式规则 |

---

## 三、交互变化对照

| 区域 | 修改前 | 修改后 |
|------|--------|--------|
| 左侧导航点击 | `switchDistrict()` + `history.pushState` 无刷新跳转 detail.html | 直接 `<a href>` 跳转 `district.html?name=xxx` |
| 移动端标签点击 | `switchDistrict()` 无刷新跳转 | 直接 `<a href>` 跳转 `district.html?name=xxx` |
| 右侧推荐卡片布局 | 背景图 + 蓝色遮罩 + 白字标题/地址 | 左侧图片 + 右侧标题 + 介绍（黑字白底） |
| 推荐分页 | 4 条/页，有上一页/下一页按钮 | 固定 5 条，无分页 |
| 推荐数据格式 | `{ id, name, address, cover }` | 与 district.html 一致（见下方） |

## 四、数据格式对齐

### district.html 原始 ktvData 字段

```js
{
    id: Number,          // 门店 ID
    title: String,       // 门店标题
    time: String,        // 发布日期
    address: String,     // 地址
    package: String,     // 套餐信息
    description: String, // 门店描述
    phone: String        // 电话
}
```

### detail.js 推荐数据字段（已对齐）

```js
{
    id: String,          // 门店 ID
    title: String,       // 门店标题（对应 district.html 的 title）
    time: String,        // 发布日期
    address: String,     // 地址
    package: String,     // 套餐信息
    description: String, // 门店描述（用于推荐卡片展示）
    phone: String,       // 电话
    cover: String        // 图片 URL（后端传参时与 district.html 一致）
}
```

> 接入真实 API 后，推荐接口返回的数据结构应与列表页 `ktvData` 完全一致，仅额外增加 `cover` 字段用于卡片封面图。

## 五、左侧导航链接变更

### 修改前

```html
<a href="detail.html?district=qingxiu" data-district="qingxiu">青秀区</a>
```

点击后 JS 拦截 `e.preventDefault()`，调用 `switchDistrict()` 无刷新切换。

### 修改后

```html
<a href="district.html?name=qingxiu" data-district="qingxiu">青秀区</a>
```

直接浏览器跳转至城区列表页，无 JS 拦截。

### 移动端标签栏同步修改

```html
<a href="district.html?name=qingxiu" data-district="qingxiu">青秀区</a>
```

## 七、JS 逻辑变更

### 移除的函数/变量

| 名称 | 说明 |
|------|------|
| `switchDistrict()` | 不再需要无刷新地区切换 |
| `recommendPage` | 不再需要分页状态 |
| `recommendPageSize` | 不再需要每页条数 |
| `recommendTotal` | 不再需要总条数 |
| 分页按钮事件监听 | 已移除 |
| `popstate` 事件监听 | 已移除（无 pushState） |

### 新增/修改的函数

| 名称 | 说明 |
|------|------|
| `fetchRecommend(district, excludeId)` | 参数从 `(district, page, pageSize)` 改为 `(district, excludeId)`，固定返回 5 条 |
| `renderRecommend(list)` | 参数从 `{ list, total, page, pageSize }` 简化为 `list` 数组，渲染标题+介绍+图片 |
| `loadRecommend(district, excludeId)` | 简化，不再传页码 |

### 推荐卡片渲染模板

```js
'<a href="detail.html?district=' + currentDistrict + '&id=' + item.id + '" class="recommend-card">' +
    imgHtml +                                              // <img> 或占位 div
    '<div class="recommend-card-body">' +
        '<div class="recommend-card-title">' + escapeHtml(item.title) + '</div>' +
        '<div class="recommend-card-desc">' + escapeHtml(item.description) + '</div>' +
    '</div>' +
'</a>'
```

## 八、后端 API 对接指南

### 门店详情接口

```
GET /api/ktv/detail?id=xxx&district=xxx
```

返回字段应包含：`id, title, time, address, package, description, phone, images[], district`

### 推荐门店接口

```
GET /api/ktv/recommend?district=xxx&count=5&excludeId=xxx
```

返回字段应与 `district.html` 列表页的 `ktvData` 一致，额外增加 `cover` 字段：

```json
{
    "code": 200,
    "data": {
        "list": [
            {
                "id": "2",
                "title": "青秀区 XX 娱乐场所（2号店）",
                "time": "2026-05-02",
                "address": "青秀区 XX 路 XX 号 XX 大厦",
                "package": "XX 套餐：XX 小时欢唱 + XX 饮品",
                "description": "环境 XX 优雅，音响品质 XX，XX 聚会首选场所。",
                "phone": "138xxxx2000",
                "cover": "https://xxx/cover2.jpg"
            }
        ]
    }
}
```

> `excludeId` 参数用于排除当前正在查看的门店，避免推荐重复。

## 九、响应式适配

| 断点 | 推荐卡片适配 |
|------|-------------|
| PC（>1024px） | 横排：图片 100×90 + 右侧标题/介绍 |
| 平板（768-1024px） | 横排：图片 100×90 + 右侧标题/介绍 |
| 手机（480-768px） | 横排：图片 110×80 + 右侧标题/介绍 |
| 小手机（<480px） | 横排紧凑：图片 80×65 + 右侧标题/介绍（介绍限 1 行） |
