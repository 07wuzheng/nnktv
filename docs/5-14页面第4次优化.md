## 一、修复 float-wechat-qr 二维码溢出适配问题

### 问题描述
`class="float-wechat-qr"` 容器内的二维码图片和占位文字同时作为 flex 子元素排列，导致图片溢出容器边界。

### 修复内容

**CSS (`css/style.css`)**

| 属性 | 修改前 | 修改后 |
|------|--------|--------|
| `.float-wechat-qr` | 无 `overflow` | 添加 `overflow: hidden` |
| `.float-wechat-qr img` | `width:100%; height:100%; object-fit:contain` | 新增 `display: block` 防止行内元素底部间隙 |
| `.qr-placeholder-text` | `text-align:center; line-height:1.4` | 改为 `position:absolute; inset:0; display:flex; align-items:center; justify-content:center`（绝对定位覆盖在图片上方，图片加载成功时自动隐藏） |
| 新增规则 | 无 | `.float-wechat-qr img:not([src=""]):not([src="about:blank"]) ~ .qr-placeholder-text { display: none; }`（图片有有效 src 时隐藏占位文字） |
| 480px 断点 | `.float-wechat-qr { right: 40px; }` | 移除 `right: 40px`，继承 768px 断点的 `right: 100%` 左侧弹出定位 |

### 影响页面
全部 8 个页面（index / detail / district / articles / article-detail / cooperation / partnership / lianxiwomen）

## 二、业务逻辑确认

### 图片数据统一来源

| 位置 | 图片字段 | 数据来源 | 说明 |
|------|----------|----------|------|
| 首页7个地区卡片背景图 | `cover` | 该区**第一家门店**的封面图 | 后端接口 `GET /api/ktv/first-store-covers` |
| 热门KTV门店封面图 | `cover` | 门店详情页主图（同一张） | 后端接口 `GET /api/ktv/featured?limit=8` |
| 门店详情页主图 | `cover` | 门店自身封面图 | 后端接口 `GET /api/ktv/detail` |
| 地区列表页门店封面图 | `cover` | 同门店详情页图（同一张） | 后端接口 `GET /api/ktv/list` |
| 最新资讯封面图 | `thumbnail` | 文章详情页文章图（同一张） | 后端接口 `GET /api/articles?type=news&limit=3` |
| 资讯列表页封面图 | `thumbnail` | 同文章详情页图（同一张） | 后端接口 `GET /api/articles` |
| 文章详情页文章图 | `thumbnail` | 文章自身缩略图 | 后端接口 `GET /api/articles/detail` |

**核心原则**：后端统一返回 `cover`（门店）/ `thumbnail`（资讯）字段，前端多处展示共用同一图片 URL，按展示尺寸通过 CSS `background-size: cover` 等比缩放。

### 热门KTV门店排序逻辑

- **排序规则**：按每个地区**最新上传**的门店优先展示（`create_time DESC`）
- **接口参数**：`GET /api/ktv/featured?limit=8`
- **新增字段**：`create_time`（门店创建时间，用于排序）

### 代码变更

**`js/main.js`**
- `fetchFeaturedStores()` 注释更新：补充排序逻辑说明和 `create_time` 字段
- 模拟数据新增 `create_time` 字段
- `loadDistrictCovers()` 注释更新：补充图片逻辑说明
- API 文档字符串统一更新

**`index.html`**
- 地区卡片 HTML 注释更新：`first-store?district=` → `first-store-covers`
- 热门KTV门店 HTML 注释更新：补充排序逻辑和 `create_time` 字段
- 最新资讯 HTML 注释更新：补充图片共用说明
