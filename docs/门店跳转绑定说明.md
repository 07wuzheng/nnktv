# 南宁KTV 门店跳转绑定说明

## 概述

南宁KTV网站各页面之间已建立门店跳转绑定关系，用户在任何页面点击门店即可跳转到对应的详情页。所有跳转链接遵循 **URL 不含中文** 的规则，使用拼音 slug 标识地区。

---

## 跳转链路总览

```
首页 (index.html)
  │
  ├── 地区卡片 → district.html?name=qingxiu      （地区列表页）
  │
  ├── 热门门店卡片 → detail.html?district=qingxiu&id=1  ★ 核心跳转
  │
  └── 文章标题（关联门店时）→ detail.html?district=xxx&id=xxx

详情页 (detail.html)
  │
  ├── 左侧地区导航 → 切换地区（pushState 无刷新，更新详情内容）
  │
  ├── 右侧推荐门店卡片 → detail.html?district=xxx&id=xxx  ★ 核心跳转
  │
  └── 面包屑 → district.html?name=xxx / index.html
```

---

## URL 参数规范

### 详情页 URL 格式

```
detail.html?district={拼音slug}&id={门店ID}
```

| 参数 | 说明 | 示例 |
|------|------|------|
| `district` | 门店所属城区拼音 | `qingxiu`、`xixiangtang` |
| `id` | 门店唯一标识 | `1`、`2`、`3` |

### 地区列表页 URL 格式

```
district.html?name={拼音slug}
```

## 首页 → 详情页绑定

### 热门门店区块

**位置**：首页 Banner 与文章列表之间（`#featuredStores` 区块）

**数据来源**：
```
GET /api/ktv/featured?limit=8
```

**返回格式**：
```json
{
  "code": 200,
  "data": {
    "list": [
      {
        "id": "1",
        "name": "青秀区XX KTV",
        "district": "qingxiu",
        "address": "南宁市青秀区xx路xx号",
        "cover": "https://example.com/cover.jpg"
      }
    ]
  }
}
```

**渲染逻辑**（`main.js` → `renderStoreCards()`）：
- 每个门店生成一个 `<a class="store-card">` 链接
- `href` 格式：`detail.html?district={item.district}&id={item.id}`
- `data-district` 属性存储地区拼音，便于后续筛选
- `data-id` 属性存储门店 ID，便于后续追踪
- 封面区域：有 `cover` URL 时用背景图，否则灰色占位
- 城区标签：左上角浮层显示区名

**当前状态**：使用硬编码模拟数据，已标注 `// TODO: 接入真实 API`

### 文章列表

**位置**：首页底部（`#articleList` 区块）

**跳转规则**：
- 若文章对象包含 `district` + `ktvId` 字段，标题渲染为 `<a>` 链接
- 链接格式：`detail.html?district={article.district}&id={article.ktvId}`
- 无关联门店的文章，标题为普通文本

## 详情页 → 详情页绑定

### 右侧推荐门店

**位置**：详情页右侧栏（`.recommend-list`）

**数据来源**：
```
GET /api/ktv/recommend?district={district}&page=1&pageSize=4
```

**跳转逻辑**（`detail.js` → `renderRecommend()`）：
- 每个推荐门店生成 `<a class="recommend-card">`
- `href` 格式：`detail.html?district={currentDistrict}&id={item.id}`
- 点击后跳转到新门店的详情页（整页刷新）
- 当前门店卡片添加 `.current` 类高亮标识

### 左侧地区导航

**位置**：详情页左侧栏（`.detail-sidebar-left`）

**切换逻辑**（`detail.js` → `switchDistrict()`）：
- 使用 `data-district` 属性标识地区
- 点击后 `e.preventDefault()` 阻止默认跳转
- `history.pushState()` 无刷新更新 URL
- 重新调用 `fetchDetail()` 和 `fetchRecommend()` 更新页面内容
- 支持 `popstate` 事件，浏览器前进/后退正常工作

## API 对接指南

### 1. 热门门店接口（首页）

```javascript
// main.js — fetchFeaturedStores() 函数

// 修改前（静态模拟）：
function fetchFeaturedStores() {
    var mockList = [ ... ];
    return Promise.resolve({ code: 200, data: { list: mockList } });
}

// 修改后（真实 API）：
function fetchFeaturedStores() {
    return fetchJSON('/api/ktv/featured?limit=8')
        .then(function(json) {
            if (json && json.code === 200) return json;
            return null;
        });
}
```

**必须字段**：`id`、`name`、`district`（拼音）、`address`、`cover`（可选）

### 2. 门店详情接口（详情页）

```javascript
// detail.js — fetchDetail() 函数

// 修改前：
function fetchDetail(id, district) {
    return Promise.resolve({ ... });
}

// 修改后：
function fetchDetail(id, district) {
    return fetch('/api/ktv/detail?id=' + encodeURIComponent(id) + '&district=' + encodeURIComponent(district))
        .then(function(res) { return res.json(); })
        .then(function(json) {
            if (json.code === 200) return json.data;
            throw new Error('API error: ' + json.code);
        });
}
```

**必须字段**：`id`、`title`、`district`（拼音）、`address`、`date`、`content`、`images`、`phone`（可选）

### 3. 推荐门店接口（详情页右侧）

```javascript
// detail.js — fetchRecommend() 函数

// 修改前：
function fetchRecommend(district, page, pageSize) {
    return Promise.resolve({ ... });
}

// 修改后：
function fetchRecommend(district, page, pageSize) {
    return fetch('/api/ktv/recommend?district=' + encodeURIComponent(district) + '&page=' + page + '&pageSize=' + pageSize)
        .then(function(res) { return res.json(); })
        .then(function(json) {
            if (json.code === 200) return json.data;
            throw new Error('API error: ' + json.code);
        });
}
```

**必须字段**：`list[].id`、`list[].name`、`list[].address`、`list[].cover`（可选）、`total`、`page`、`pageSize`

### 4. 文章列表接口（首页）

现有接口 `/api/articles?type=news` 返回的 `list` 数组中，每个 article 对象新增可选字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `district` | string | 关联门店所属地区拼音（如 `qingxiu`） |
| `ktvId` | string | 关联门店 ID（如 `1`） |

当这两个字段同时存在时，文章标题自动渲染为可点击链接，跳转到关联门店的详情页。

## 跳转参数传递规范

### 首页 → 详情页

```
门店卡片点击
  → href="detail.html?district=qingxiu&id=1"
  → detail.js 读取 URL 参数
  → fetchDetail('1', 'qingxiu')
  → renderDetail(data)
```

### 详情页 → 详情页（推荐切换）

```
推荐卡片点击
  → href="detail.html?district=qingxiu&id=2"
  → 整页跳转，detail.js 重新初始化
  → fetchDetail('2', 'qingxiu')
  → renderDetail(data)
```

### 详情页 → 详情页（地区切换，无刷新）

```
左侧导航点击
  → e.preventDefault()
  → switchDistrict('xingning')
  → history.pushState({}, '', 'detail.html?district=xingning')
  → fetchDetail('1', 'xingning')  // 取该区默认门店
  → fetchRecommend('xingning', 1, 4)
  → renderDetail + renderRecommend
```

## SEO 注意事项

1. **URL 禁止中文**：所有 `district` 参数使用拼音 slug
2. **语义化链接**：门店卡片使用 `<a>` 标签，搜索引擎可跟踪
3. **data 属性**：`data-district`、`data-id` 辅助 JS 交互，不影响 SEO
4. **内链网络**：首页门店 → 详情页、详情页推荐 → 详情页，形成内链闭环
5. **结构化数据**：详情页 `LocalBusiness` 类型包含 `url` 字段指向正确的拼音 URL
