# 南宁KTV详情页优化说明

## 优化内容

1. **地址字段**：详情区新增 `<address>` 标签展示门店地址，语义化更好，利于搜索引擎识别。
2. **左侧地区导航**：垂直显示南宁 7 区，点击切换地区时**停留在详情页**，中间内容切换为该地区门店，推荐列表同步更新。使用 `history.pushState()` 无刷新切换。
3. **右侧推荐分页**：卡片式推荐门店，尺寸加大（140px 高），采用图片背景模拟 + 遮罩 + 白色文字，底部提供"上一页/下一页"分页按钮。
4. **联系我们按钮**：页面底部居中，类名 `btn-phone` + `detail-contact-btn`，点击弹出电话弹窗（与 `district.html` 一致）。
5. **固定居中布局**：左右侧栏使用 `position: sticky` 定位，三栏整体水平居中，滚动时侧栏保持可见。

## 布局结构

- **桌面端（>768px）**：Flex 容器水平排列 `[左导航 220px] [中间内容 flex:1, max-width:800px] [右推荐 300px]`，左右 `position: sticky; top: calc(64px+24px); align-self: flex-start;`。

  - 左导航变为顶部横向滚动标签栏（`position: sticky; top: 64px;`）
  - 右推荐移至正文下方，取消固定定位
  - 中间详情区宽度 100%，padding 16px
  - 联系我们按钮宽度 100%

## 地区切换逻辑

### 交互流程

```
用户点击左侧地区导航（如"兴宁区"）
    │
    ├── e.preventDefault() — 阻止默认跳转
    │
    ├── history.pushState() — 更新 URL：detail.html?district=xingning
    │
    ├── 更新变量：districtSlug = 'xingning', districtName = '兴宁区'
    │
    ├── 高亮左侧当前地区导航项 + 移动端标签
    │
    ├── fetchDetail('1', 'xingning') → renderDetail(data)
    │   ├── 更新标题、地址、正文内容
    │   ├── 更新 SEO 元数据（title、meta、canonical、OG、结构化数据）
    │   └── 更新面包屑导航
    │
    └── loadRecommend(1) → 渲染推荐卡片 + 重置分页
```

### 关键实现

- 左侧链接使用 `data-district` 属性存储拼音标识
- 点击后 `e.preventDefault()`，调用 `switchDistrict(newDistrict)` 函数
- `history.pushState()` 更新浏览器 URL，不刷新页面
- 支持 `popstate` 事件，浏览器前进/后退也可正常切换

## 推荐分页对接

### API 地址

```
GET /api/ktv/recommend?district=xxx&page=1&pageSize=4
```

### 返回格式

```json
{
    "code": 200,
    "data": {
        "list": [
            { "id": 2, "name": "xx", "address": "xx", "cover": "url" },
            { "id": 3, "name": "xx", "address": "xx", "cover": "url" }
        ],
        "total": 20,
        "page": 1,
        "pageSize": 4
    }
}
```

### 前端实现

- 维护 `recommendPage`（当前页码）和 `recommendPageSize`（每页 4 条）
- 点击"上一页/下一页"时调用 `loadRecommend(page)`
- `fetchRecommend()` 中已注释真实 API 调用，替换 TODO 即可对接
- 分页控件自动计算总页数，禁用边界按钮

## SEO 要点

1. **标题格式**：`门店名 - 南宁KTV`
2. **描述含地址、地区信息**：`xx，地址xx，南宁热门KTV推荐`
3. **所有内部链接使用拼音路径**，禁止中文编码
4. **结构化数据**：
   - `LocalBusiness` 类型：包含地址、名称、图片等，待动态填充
   - `BreadcrumbList`：首页 → 城区 KTV → 门店名
5. **`<address>` 标签**：语义化展示门店地址
6. **Canonical URL**：随地区/门店切换动态更新
7. **OG 标签**：社交分享时展示正确信息
