# 城区页门店跳转详情页 — 绑定说明

## 一、跳转链路总览

```
district.html?name=qingxiu
  └─ KTV 门店卡片（3 个点击区域）
       ├─ 卡片封面图 ──→ detail.html?district=qingxiu&id=1
       ├─ 卡片标题   ──→ detail.html?district=qingxiu&id=1
       └─ "查看详情"按钮 ──→ detail.html?district=qingxiu&id=1
       └─ "查看电话"按钮 ──→ 弹窗显示电话（不跳转）
```

## 二、URL 参数规范

| 参数 | 说明 | 示例 | 来源 |
|------|------|------|------|
| `district` | 城区拼音 slug（URL 禁止中文） | `qingxiu` | 侧边栏 `name` 参数 |
| `id` | 门店 ID（数字） | `1` | 门店数据 `id` 字段 |

### 城区拼音对照表

| 拼音 slug | 中文名 |
|-----------|--------|
| `qingxiu` | 青秀区 |
| `xingning` | 兴宁区 |
| `jiangnan` | 江南区 |
| `xixiangtang` | 西乡塘区 |
| `liangqing` | 良庆区 |
| `yongning` | 邕宁区 |
| `wuming` | 武鸣区 |

### URL 生成逻辑

```javascript
var detailUrl = 'detail.html?district=' + encodeURIComponent(slug) + '&id=' + encodeURIComponent(item.id);
```

- `slug`：从 URL 参数 `?name=xxx` 获取，如 `qingxiu`
- `item.id`：门店数据中的 `id` 字段，如 `1`

## 三、修改内容

### 3.1 district.html — renderPage 函数

**修改前**：门店卡片无跳转链接，只有"查看电话"按钮。

**修改后**：门店卡片有 3 个可点击跳转区域 + 1 个电话弹窗按钮：

| 点击区域 | 跳转目标 | HTML 元素 |
|----------|----------|-----------|
| 卡片封面图 | `detail.html?district=xxx&id=xxx` | `<a class="ktv-card-link">` |
| 卡片标题 | `detail.html?district=xxx&id=xxx` | `<h2><a>` |
| "查看详情"按钮 | `detail.html?district=xxx&id=xxx` | `<a class="btn-detail">` |
| "查看电话"按钮 | 电话弹窗（不跳转） | `<button class="btn-phone">` |

**新增 HTML 结构**：

```html
<article class="ktv-card" data-district="qingxiu" data-id="1">
  <!-- 封面图（可点击跳转） -->
  <a href="detail.html?district=qingxiu&id=1" class="ktv-card-link">
    <div class="ktv-img">图片占位</div>
  </a>
  <div class="ktv-body">
    <!-- 标题（可点击跳转） -->
    <h2><a href="detail.html?district=qingxiu&id=1">青秀区 XX 娱乐场所（1号店）</a></h2>
    <p class="ktv-meta">发布日期：2026-05-01</p>
    <div class="ktv-info">
      <p><span class="label">地址：</span>...</p>
      <p><span class="label">套餐：</span>...</p>
    </div>
    <p class="ktv-desc">...</p>
    <!-- 按钮组 -->
    <div class="ktv-actions">
      <a href="detail.html?district=qingxiu&id=1" class="btn-detail">查看详情</a>
      <button class="btn-phone" data-phone="138xxxx0001">查看电话</button>
    </div>
  </div>
</article>
```

### 3.2 district.html — 新增 CSS

| CSS 类 | 说明 |
|--------|------|
| `.ktv-card-link` | 封面图点击区域，`display:block` 铺满图片区 |
| `.btn-detail` | "查看详情"按钮，主色实心背景，hover 变深+微上浮 |
| `.ktv-actions` | 按钮组 flex 容器，`gap:12px` |
| `.ktv-body h2 a` | 标题链接，默认深色，hover 变主色 |

### 3.3 data 属性（便于后期 API 对接）

每张门店卡片添加了 `data-district` 和 `data-id` 属性：

```html
<article class="ktv-card" data-district="qingxiu" data-id="1">
```

- 后期可通过 JS 选取：`document.querySelectorAll('.ktv-card[data-district="qingxiu"]')`
- 便于事件委托、埋点统计、动态更新等场景

## 五、API 对接指南

### 5.1 城区页门店列表 API

当前使用 `district.html` 内联脚本中的模拟数据。对接真实 API 时：

**修改位置**：`district.html` 内联脚本第 4 节"模拟数据"

**替换步骤**：

1. 将模拟数据生成逻辑替换为 API 调用：

```javascript
// 修改前（模拟数据）
var ktvData = [];
for (var i = 1; i <= 12; i++) {
    ktvData.push({ id: i, title: '...', ... });
}

// 修改后（真实 API）
async function fetchKtvList(district, page) {
    var res = await fetch('/api/ktv/list?district=' + district + '&page=' + page + '&size=6');
    var json = await res.json();
    if (json.code === 200) return json.data;
    throw new Error('API error: ' + json.code);
}
```

2. API 返回数据结构需包含以下字段：

```json
{
    "code": 200,
    "data": {
        "list": [
            {
                "id": "1",
                "title": "门店名称",
                "time": "2026-05-01",
                "address": "南宁市青秀区XX路XX号",
                "package": "XX套餐：XX小时欢唱+XX饮品",
                "description": "环境优雅，音响品质...",
                "phone": "138xxxx0001",
                "cover": "https://xxx/cover.jpg"
            }
        ],
        "total": 12,
        "page": 1,
        "pageSize": 6
    }
}
```

3. `id` 和 `district`（从 URL `name` 参数获取）将自动拼接为详情页跳转链接。

### 5.2 详情页 API

详情页 API 对接方式参见 `docs/门店跳转绑定说明.md`。

### 5.3 关键注意事项

- **`district` 参数必须使用拼音 slug**，不使用中文，确保 URL 合法
- **`id` 参数建议使用字符串类型**，避免大数字精度丢失
- API 返回的门店数据中必须包含 `id` 字段，否则无法构建详情页链接
- 封面图 URL 存入 `cover` 字段，渲染时判断是否有有效 URL 来决定显示图片或占位
