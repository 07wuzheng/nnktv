# 南宁 KTV 详情页模板 — 使用说明

## 一、页面功能概述

`detail.html` 是南宁 KTV 网站的门店详情页模板，用于展示单家 KTV 的推广文章内容。

**核心功能：**
- 通过 URL 参数 `id` 接收门店 ID，动态加载详情数据
- 展示门店标题、地址、发布日期、正文内容、多张配图
- 左侧城区导航栏（sticky 固定，不随页面滚动）
- 右侧同区门店推荐栏（sticky 固定，不随页面滚动）
- 面包屑导航：首页 → 城区 KTV → 门店名称
- 完整的 SEO 元数据（title、description、keywords、canonical、OG 标签）
- 结构化数据（Article + BreadcrumbList）
- 响应式设计，适配桌面端与移动端
- 图片占位块：后端返回真实图片 URL 后自动替换为 `<img>` 标签

**URL 格式：**
```
detail.html?id=123&district=qingxiu
```
- `id`：门店 ID（必填）
- `district`：所属城区拼音（可选，用于左侧导航高亮、面包屑和返回链接）

## 三、如何对接后端 API

### 3.1 详情数据 API

| 项目     | 说明                          |
|:---------|:------------------------------|
| 请求方法 | `GET`                         |
| URL      | `/api/ktv/detail?id=xxx`      |
| 返回格式 | JSON                          |

**返回 JSON 示例：**

```json
{
  "code": 200,
  "data": {
    "id": 1,
    "title": "青秀区 XX 量贩 KTV（1号店）",
    "date": "2026-05-01",
    "address": "南宁市青秀区 XX 路 XX 号 XX 大厦",
    "content": "本店位于青秀区核心地段，环境优雅，音响品质一流，是朋友聚会、商务宴请的首选场所。\n店内提供多种套餐选择，从小型包厢到豪华大包应有尽有，满足不同需求。\n所有包厢均配备专业级音响设备和智能点歌系统，让您尽情欢唱。\n欢迎来电咨询预订，我们竭诚为您服务。",
    "images": [
      "https://example.com/ktv1/photo1.jpg",
      "https://example.com/ktv1/photo2.jpg",
      "https://example.com/ktv1/photo3.jpg"
    ]
  }

```

### 3.2 推荐门店 API

| 项目     | 说明                                        |
|:---------|:--------------------------------------------|
| 请求方法 | `GET`                                       |
| URL      | `/api/ktv/recommend?district=xxx&excludeId=xxx` |
| 返回格式 | JSON                                        |

**返回 JSON 示例：**

```json
{
  "code": 200,
  "data": [
    { "id": 2, "title": "青秀区 XX KTV（2号店）", "address": "南宁市青秀区 XX 路", "date": "2026-05-02" },
    { "id": 3, "title": "青秀区 XX KTV（3号店）", "address": "南宁市青秀区 XX 路", "date": "2026-05-03" },
    { "id": 4, "title": "青秀区 XX KTV（4号店）", "address": "南宁市青秀区 XX 路", "date": "2026-05-04" }
  ]
}
```

### 3.3 对接步骤

1. 打开 `detail.html`，找到 `fetchDetail` 函数
2. 取消注释 `fetch()` 调用代码，删除 `Promise.resolve()` 模拟数据
3. 找到 `fetchRecommend` 函数，同理替换
4. 确认返回的 JSON 结构与上述示例一致
5. 如字段名不同，调整 `renderDetail` / `renderRecommend` 函数中的字段映射

```javascript
// 修改前（静态模拟）：
function fetchDetail(id) {
    return Promise.resolve({ id: id, title: 'xx', ... });
}

// 修改后（真实 API）：
function fetchDetail(id) {
    return fetch('/api/ktv/detail?id=' + encodeURIComponent(id))
        .then(function(res) { return res.json(); })
        .then(function(json) {
            if (json.code === 200) return json.data;
            throw new Error('API error: ' + json.code);
        });
}
```

## 四、需替换的占位符列表

| 占位内容           | 位置                          | 替换说明                                     |
|:-------------------|:------------------------------|:---------------------------------------------|
| `xx`（标题）       | `<h1>`、`<title>`、OG 标签    | 替换为门店真实名称                           |
| `xx`（地址）       | `<span id="detailAddress">`   | 替换为门店真实地址                           |
| `xx`（日期）       | `<time>`、结构化数据          | 替换为真实发布日期，如 `2026-05-12`          |
| `xx`（正文段落）   | `<p>` 标签                    | 替换为真实推广文章内容                       |
| `url1.jpg` 等      | 图片占位块 `data.images`      | 替换为真实图片 URL，占位块自动变为 `<img>`  |
| `alt=""`（图片）   | `<img>` 标签                  | 替换为有意义的描述，如 "XX KTV 大厅实拍"     |
| `id=xx`（canonical）| `<link rel="canonical">`     | 替换为真实门店 ID                            |
| `xx`（description）| `<meta name="description">`  | 替换为包含门店名称和地址的真实描述           |
| `xx`（推荐门店）   | 右侧推荐列表                  | 替换为同城区其他门店的标题、地址、日期       |

## 七、SEO 优化建议

### 已实现的 SEO 措施

1. **语义化 HTML**：使用 `<article>`、`<h1>`、`<time>`、`<figure>`、`<p>`、`<aside>`、`<nav>` 等语义标签
2. **唯一 H1**：每页仅一个 `<h1>`，内容为门店名称
3. **Canonical 标签**：防止重复内容，URL 中不含中文
4. **面包屑导航**：帮助搜索引擎理解页面层级（3 级）
5. **结构化数据**：Article + BreadcrumbList 双重 JSON-LD
6. **OG 标签**：支持社交媒体分享
7. **theme-color**：移动端浏览器地址栏颜色适配
8. **无障碍**：`skip-link`、`aria-label`、`aria-current` 等
9. **地址字段**：在 meta description 和结构化数据中包含门店地址

### 后续 SEO 优化建议

1. **图片 alt 属性**：后端返回图片时同步返回 `alt` 描述文本
2. **首屏内容优先**：确保正文首段包含核心关键词
3. **内链建设**：右侧推荐列表天然形成内链网络
4. **页面加载速度**：图片使用 WebP 格式 + lazy loading
5. **sitemap.xml**：将所有详情页 URL 加入站点地图
6. **独立静态页面**：后期可为每个门店生成独立 HTML 文件，提升收录效率
7. **本地 SEO**：在结构化数据中补充 `LocalBusiness` 类型，添加地址、电话、营业时间

## 九、图片占位与动态替换机制

### 占位块 CSS 定义

```css
.img-placeholder {
    width: 100%;
    max-width: 800px;
    height: 200px;        /* 桌面端 200px，移动端 150px */
    background: #f0f0f0;  /* 灰色背景 */
    border-radius: 8px;
    margin: 0 auto;
}
```

### 动态替换逻辑

1. 后端 API 返回 `images` 数组，包含真实图片 URL
2. JS 遍历 `images`，判断 URL 是否有效：
   - 有效 URL（包含 `http` 且不包含 `url` 占位符）→ 创建 `<img>` 标签
   - 无效/占位 URL → 保留灰色 `<div class="img-placeholder">`
3. 动态创建的 `<img>` 样式：
   ```css
   .detail-content figure img {
       max-width: 100%;
       height: auto;
       display: block;
       margin: 0 auto;
       border-radius: 8px;
   }
   ```

## 十、注意事项

1. **URL 禁止中文**：所有 URL 参数、文件名、锚点必须使用拼音或数字，不得包含中文字符
2. **不使用图标字体**：本模板不引入任何图标库（FontAwesome 等），所有视觉元素仅用文字和占位块
3. **纯静态渲染**：不依赖任何前端框架或构建工具，可直接在浏览器中打开
4. **JSDoc 注释**：所有 API 调用函数均有完整的参数说明和替换指引
5. **错误处理**：API 请求失败时，页面会显示占位内容，不会白屏
6. **sticky 限制**：左右侧栏使用 `position: sticky`，父容器不能设置 `overflow: hidden`
7. **推荐门店排除自身**：`fetchRecommend` 传入 `excludeId` 参数，确保推荐列表不包含当前门店
